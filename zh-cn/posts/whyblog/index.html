为什么要写博客

September 07, 2024
杂谈
目录

    • 我为什么搭建博客

这篇文章阐述了我要坚持写博客的原因，以及我搭建博客的思路。

我们还需要博客吗？

博客，或者说 blog，是一个与当前时代格格不入的事物。博客最初于 1998 年出现，在 2000 年初正式流行，彼时还是 Web 2.0 时代。那时，国内大部分人对网络还没有概念，大众的娱乐方式还是以电视为主。而博客，是那个时代非常火爆的一种社交形式，人们习惯于在网络上发表文章来分享自己的生活、经验等，很多的博客服务提供商也如雨后春笋般争先恐后地破土而出。那时，你不需要掌握什么计算机知识，只需要注册一个账号就可以写博客，优秀的博主更是能收获一大批粉丝和追随者，成为那个时代的大 V。

然而，随着互联网的飞快发展，博客就像曾经的论坛、BBS 一样，无声无息地与我们渐行渐远。我们开始习惯使用即时通讯软件，如 QQ、微信和 web 3.0 如汪洋大海般的软件，像博客这样无法及时互动的单一性输出方式已经无法与我们快节奏的习惯相匹配了。

但博客，真的到了功成身退的时候了吗？我认为没有。就像我对一些逐渐被淘汰的事物的态度，比如京剧、相声、评书等，并不是真的要死了，而是回到了原本该有的位置，正如这两年的计算机相关专业。那么，博客的正确位置在哪里呢？

博客，作为一种内容单向流动的传播、分享形式，与当前的主流讯息传播方式相比，其实是有很大优势的。非及时性交流带给人们的也不只有交流不便，也带来了思考的深入和谨慎。在博客时代，即使有评论区，也很少有现在小红书、抖音、B 站甚至知乎等群魔乱舞穷追不舍的局面，因为博客始终是一个内容分享方式，而不是一种互动方式。就像很多公司工作时使用邮件而不是微信等即时通讯软件，博客和邮件一样，可以让我们在发布前对发布的内容进行更多的思考、提炼，阅读者也有时间进行更深层次的理解、判断。

可能你会说，邮件、博客也可以秒回，有些人也会不认真看啊。没错，不管是什么形式，我们都可以只大概看一下甚至根本不看就输出自己的情绪。但对大多数人来说，不同方式的影响是客观存在的。想象一下，当你阅读一封邮件时，尤其是需要回复的邮件时，你一定会下意识反复阅读内容，害怕一不小心就误解了文字的含义；而当我们看到一条微信消息时呢，我们想的是需要尽快回复，不管是回复老板还是朋友。而当我们读到了一篇意见相悖的博客时，我们下意识想总结、概括自己的观点并认真比较；可当我们在抖音、小红书甚至知乎评论区（现在的知乎）看到意见不同的帖子、评论时，我们一定是急不可耐地打开评论区表现一下互联网时代的“君子六艺”。我认为，之所以产生如此大的反差，其根源在于产品定位不同。邮件、博客主要的目的是单方面输出，彼此间的交流也是通过一封封邮件、一篇篇博客这些最小单元实现的，而单方面输出代表着接受者需要额外的时间来建立连接理解、消化；而类似微信、抖音这样的应用，其主要目的是消除时间和空间上的距离感。当我们在观看一个视频时，我们能切实感受到视频作者所处的环境、面对的事物，从而更快地理解视频中所传达的内容和情绪。可以说，它们的火爆正是因为它们减少了信息接受的成本，在这样的应用中，最小单元不仅仅是视频、帖子，还有评论、对话，它们组成了一个整体作为信息传播的最小单元，也就是说，我们的接受、处理信息的能力变强了。

但其实，这种能力的增强是有代价的。接受、处理信息速度的提升必然带来思考时间上的削减，我们越来越依靠第一反应甚至是本能反应来判断事情。在评论区经常会出现的一句很火的评论：“我这 40 m 的大刀收不住了”，想象一下，如果你是在阅读一篇博文，你会急着做出这样的反应吗？不会的，因为你知道即使你在评论区反驳，也只有等博主打开电脑登上账号才能被看到，而用手机刷短视频无疑占据着我们很多时间，相关作者也更期待来自评论的对话，这正是我上文提到的传播最小单元的改变带来的结果。

在这样的背景下，博客的意义是什么呢？博客可以让我们慢下来，一方面，他可以让读者慢下来，不再急于发表自己的看法而是投入更多的思考；但更重要的，也是我觉得博客可以存活下来的核心竞争力，是博客会让博主更专注内容本身而不是互动。诚然，做视频要比写博文累得多，但做视频其实是在与观众直接对话，也更期待观众的反馈；而博客则不然，博客是在把我具有的内容提炼、总结分享出来，这种单方面的流动会减少我们对于反馈的期待，从而注重如何更好地分享。在训练大语言模型时，相比于“君子六艺”，我们也更希望用一篇篇博文来训练模型，从而让大语言模型能获得更好的内容输出能力。

我为什么搭建博客

这是本文的第二个方面，为什么我要搭建博客，甚至租服务器和域名来搭建自己的博客。

原因其实比较多，但总结起来有以下几点：

  • 对内容审查的警惕；
  • 追求自我空间的私密性；

内容审查

在国内，原则上不允许个人博客网站这一形式存在。ICP 备案时客服明确说资源用途不可以出现博客两个字，不备案就不能使用，备案就意味着自己的内容处于随时被监控的状态。这种被监控的状态让我很不舒服，也无法完整表达自己的观点。我以后大概是会在国内发展的，我不想有一天因为博客内容而对前途产生影响。

自我空间的私密性

第二个原因是，我写博客的主要目的并不是给别人看，只是想有一个私人空间来存放自己的内容，或是学习心得，或是生活感悟，如果有人愿意看并且产生了一定的帮助，这是意外之喜。

相比于成熟的博客网站，个人搭建博客的另一大好处是足够私密。对于博客服务提供商，它势必要挑选潜在的明星博主来作为盈利的手段，不论是带货、广告等，这对我是一个十分负面的性质。而自己搭建博客，不论是 SEO 还是推广，都要弱很多，这恰好满足了我的需求。

或许你会问，那你为什么不直接写文章存到本地？因为相比于写博客，只在本地写文章、笔记很多时候都会流于形式甚至中道崩卒；而博客，发布后始终是可能被人看到的，自然就需要更加严谨。

博客的搭建思路

我的博客使用 Hugo 搭建，这是一个用 Go 写的静态页面生成器，速度很快。我最早尝试过使用 Wordpress 搭建动态页面，但那时对计算机还没有多少认识，服务器到期就作废了。后来还使用过 Hexo，但是我并不是很喜欢 Node.JS，所以一直在寻找其替代品，最终确定了 Hugo。

我的博客一开始部署在 GitHub Page，但是由于用 Git 上传大文件很麻烦，所以最终决定租用服务器；同时，为了免去备案，我租用了架设在东京的服务器。

我的博客没有设置评论功能，虽然主题本身支持很多评论方式，但我觉得既然写博客，不如索性不要评论，大家通过邮件联系，这样才能体现博客的价值。

我现在还是一位学生，等之后工作了，希望能自己搭建一个服务器部署博客，从而达到更好的稳定性和私密性。

—— 2024 年 9 月 7 日 四点零八分写于江安宿舍
